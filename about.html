Loops in PHP

A loop is a cyclic repetition of a process that executes a specific task. The condition is checked:

  • If the condition is true, the loop iterates.
  • The loop keeps iterating until the condition becomes false.
  • When false, the loop stops and control is sent outside the loop.

Types of Loops in PHP

  1. while loop
  2. do…while loop
  3. for loop
  4. foreach loop

While Loop Syntax

initialize the loop_variable;
while(condition) {
    // Code
    // Increment / Decrement the loop_variable
}

Notes: loop_variable is a regular variable used to create the loop condition. Initialize it with a start value.

Do-While Loop Syntax

initialize the loop_variable;
do {
    // Code
    // Increment / Decrement the loop_variable
} while(condition);

For Loop Syntax

for(initialization; condition; increment/decrement) {
    // Code
}

Arrays

→ An array is a special variable.

→ An array can store multiple values in a single variable.

→ Three types of arrays:

  • Indexed array
  • Associative array
  • Multidimensional array

Indexed Array

→ Syntax for empty indexed array declaration:

$array_name = array();

→ Syntax to declare an indexed array with values:

$array_name = array(value1, value2, value3);

→ Every array element has a unique index: min index = 0 & max index = size – 1

→ Access an array element using: $array_name[index]

→ Get the size of the array using: count($array_name)

→ Display the whole array using: print_r($array_name)

→ Access all array elements in a loop:

for($index_var = 0; $index_var < count($array_name); $index_var++) {
    echo $array_name[$index_var];
}

Indexed Array Example 1

<?php
// Declare an indexed array with values
$cities = array("New Delhi", "Mumbai", "Chennai", "Kolkatta");

// Get the array size
echo("<br>Total Cities: " . count($cities));
echo("<br>");

// Display the whole array
print_r($cities);

// Access individual array elements
echo("<br>City at index 0: " . $cities[0]);
echo("<br>City at index 2: " . $cities[2]);

// Access all elements using a loop
echo("<ul>");
for($i = 0; $i < count($cities); $i++) {
    echo("<li>".$cities[$i]."</li>");
}
echo("</ul>");
?>

The __construct function

→ Special function called automatically when an object is created.

  • Name fixed: __construct()
  • May / may not take arguments
  • Cannot return a value
  • Only one constructor allowed per class
  • Can pass arguments to set data while creating object

Inheritance in PHP

→ Inheritance is creating a new class and deriving properties from a previously created class.

Benefits:

  • Reusability of code
  • Reduces code size, effort, time, and complexity

Types of Inheritance:

  1. Single Level
  2. Multi-Level
  3. Hierarchical

Single Level Inheritance

class SuperClass { } 
class SubClass extends SuperClass { }

Notes:

  • Private members of SuperClass cannot be accessed in SubClass
  • Protected members accessible in SubClass but not outside
  • Public members accessible in SubClass

Multi-Level Inheritance

class SuperClass { } 
class IntermediateSuperClass extends SuperClass { } 
class SubClass extends IntermediateSuperClass { }

Hierarchical Inheritance

class SuperClass { } 
class SubClass1 extends SuperClass { } 
class SubClass2 extends SuperClass { }

Note: PHP does not support multiple inheritance directly:

class SubClass extends SuperClass1, SuperClass2 { }

State Management

  • → By default, data of a page is not sharable across pages.
  • → State management helps share data across pages without forms.
  • → Technologies: Query String, Sessions, Cookies

Query String

  • Transports limited string data from source page to target page
  • Type & quantity: Strings up to 4kb
  • Storage: Client Browser's address bar
  • Security: Insecure
  • Scope: Available till target page runs
  • Syntax (single): TargetPageURL?key=value
  • Syntax (multiple): TargetPageURL?key1=value1&key2=value2
  • Retrieve in target page: $var = $_GET["key"];

Sessions

  • → PHP Sessions are used to store and pass information from one page to another temporarily.
  • → Sessions are widely used for sign in and shopping carts.
  • → Session is browser-specific; every browser gets a unique session id.
  • → Properties of Sessions:
    • Type & Quantity: Any data type and any size
    • Storage Location: Server’s temporary memory
    • Security: Secure
    • Scope: Accessible throughout the website
    • Life: Available only while the session is alive
  • → Session destruction cases:
    • User initiated (logout)
    • Browser session ends

Sessions Syntax

  • Start session at the top of every PHP file: session_start();
  • Create/assign session data: $_SESSION["session_key"] = value;
  • Retrieve session data: $var = $_SESSION["session_key"];
  • Destroy session: 
    // remove all session variables
session_unset();
// destroy the session
session_destroy();

    Interfaces

    • → Interfaces cannot have properties; only public member methods can be declared.
    • → Interfaces cannot be instantiated.

    interface InterfaceName {
    public function function_name($arguments);
}

class ClassName implements InterfaceName {

}

    Abstract Member Functions

    • → Abstract member functions are special functions declared in the parent class only.
    • → They cannot be defined in the parent class.
    • → Must be prefixed with abstract keyword.
    • → Can be public or protected.
    • → Syntax: abstract access_specifier function member_function_name($args);
    • → All derived classes must define these functions with the same signature.

    Abstract Class

    • → Abstract class is a parent class containing at least one abstract member function.
    • → Declared with abstract keyword.
    • → Cannot create an object of abstract class directly.
    • → Can contain properties and regular member functions.

    Example: Employee Abstract Class

    <?php
abstract class Employee {
    public $empid;
    public $empname;
    public $workinghours;
    public $workingdays;

    public function setEmpData($empid, $empname, $workinghours, $workingdays) {
        $this->empid = $empid;
        $this->empname = $empname;
        $this->workinghours = $workinghours;
        $this->workingdays = $workingdays;
    }

    abstract public function calculateSalary();
}
?>

require_once("Employee.php");

    public function calculateSalary() {

    Polymorphism

    • → Poly – many, Morph – forms.
    • → Allows objects of different classes to have same signature methods.
    • → Implemented via:
      • Abstract Classes
      • Interfaces

    Polymorphism using Abstract Class

    <?php
abstract class Person {
    abstract public function greet();
}

class English extends Person {
    public function greet() { return "Hello!"; }
}

class German extends Person {
    public function greet() { return "Hallo!"; }
}

class French extends Person {
    public function greet() { return "Bonjour!"; }
}

// usage
$person_eng = new English();
echo "<br/>English Person says: ".$person_eng->greet();
?>

    Polymorphism using Interface

    <?php
interface Greeting {
    public function greet();
}

class English implements Greeting {
    public function greet() { return "Hello!"; }
}

class German implements Greeting {
    public function greet() { return "Hallo!"; }
}

class French implements Greeting {
    public function greet() { return "Bonjour!"; }
}

// usage
$person_eng = new English();
